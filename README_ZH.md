# 传统生存型俄罗斯方块AI

[English Version](README.md)

## 简介

Misakamm 实现的一个AI，在典型俄罗斯方块规则下，**不考虑 next**，平均消行数约为 **2.7亿**，为首个(在公开纪录里)在此规则下破亿的AI。

## 规则

1. 游戏池尺寸宽10，高20 （可以用其它高度跑数据，数据也需要注明所用的高度），游戏池最大高度为死亡线
2. 每个方块从 7 个标准 tetrominoes 中无状态独立线性随机生成，无状态即与历史方块序列和当前游戏池状态无关
3. 块的出生点在游戏池上方以外，高于游戏池的死亡线，不存在因下方游戏池的方块阻挡导致无法选择某个横坐标与旋转状态的情况
4. 仅硬降无软降，即只能一降到底，不许横插操作
5. 摆放后，如果存在任何一格高于死亡线，立即结束（消行前做判定）
6. 无 next，只能考虑当前方块

### 无旋转系统

在此规则下，方块的初始坐标不再重要，因为左右平移也不可能存在被阻挡的情况，初始朝向同样也不重要。也因为无软降，也不需要考虑旋转前与旋转后的变化来“卡”进某些位置，只需要考虑某个方向在某个横坐标摆放，所以称为**无旋转系统**。而且在此规则下，AI需要处理的东西最少，最适合做AI比较，同时也是较多的方块AI论文所使用的规则。

### 积分计算

以总消行数计算单局积分。多局统计时，统计以下数据：
- 全平均
- 中位数
- 中位数除以 ln2，此数值是指数分布下的理论均值，与平均值越接近，越说明样本充足。反之可以在样本数不足的时候，使用此值估算平均值

### 随机数

本项目使用 PCG 随机数库里面的 `pcg32_k2_fast` 实现，状态空间 2^128，能通过全部已知的随机性测试，强度足够。

具体生成随机块时，首先把随机数引擎生成的32bit整数，作为一个bit stream，然后每次取 3bit，如果结果是7，那就舍去此值，取下一个 3bit，直到数值范围在 [0,6]。

## AI 评估实现

### 权重

- `row_transitions` = 100
- `column_transitions` = 95
- `hole` = 401
- `landing_height` = 67
- `rows_cleared` = 86
- `well_depth_2` = 176
- `well_depth_3` = 99
- `x1_hole` = 88
- `x2_hole` = 88
- `hole_depth` = 75
- `shape_adaptability` = 117

注：源代码中，`row_transitions` 直接写成常数，`column_transitions` 使用 `vertical`，其它的参数名字与源代码相同。

### 解释

与 Pierre Dellacherie 的算法相同的部分有：

- `row_transitions`：扣分项，每行的变换数，初始和结束都用非0值，即边界视为有方块
- `column_transitions`：扣分项，每列的变换数，从上到下计算，初始0，结束非0，即顶部视为无方块，底部有方块
- `hole`：扣分项，每一个洞的所扣的分

其它的参数：

- `landing_height`：正分项，摆放好后，摆放的方块的那4小格，距离顶部的格数的均值
- `rows_cleared`：正分项，消除的行数
- `well_depth_2`：扣分项，当某列高度比两边都要矮时，两边高度差取较小值为 h，接着按 h 分类：如果 h 是 1，那么值为 `well_depth_2 / 5`；如果 h 是 2，那么值为 `well_depth_2`；如果 h 大于等于 3，那么值为 `well_depth_3 + (h - 2) * hole`，其中，`hole` 的值取决于横坐标，表示该列的洞的扣分分值。比如贴着游戏池最左或最右的情况时，分值为 `hole + x0_hole`。然后记录所有井的值的和 sum，以及单个井的最大值 max，最后是 `sum * 2 - max`
- `x1_hole`：扣分项，当某个洞的横坐标，是距离游戏池最左或最右1格的情况下，额外扣的分。类似地，还有 `x0_hole` 和 `x2_hole`，其中 `x0_hole` 的值固定为 `x1_hole` 的 1.5 倍，是贴着游戏池最左或最右的情况，`x2_hole` 是距离游戏池最左或最右2格的情况
- `hole_depth`：扣分项，找到最高的那个洞，计算它所在列的高度。如果有同时多个最高的洞，分别取它们所在列的高度，再求它们的最大值
- `shape_adaptability`：正分项，这个项最为复杂，扫描所有的连续两列以及连续三列，比如说，高度依次为 `3, 2, 3`，能匹配 T 的形状，就给这个形状匹配数加 1。如果高度依次为 `3, 3, 3`，就能匹配 T 上下倒置的方向，以及 L 和 J 平放，同理这个 3 形状分别加 1。然后对于每个形状，匹配数量从 0 到 2，大于 2 也算 2，对应的分数表是 `0, 4, 6`，即不匹配就 0 分，匹配 1 次就 4 分。最后，去掉 I 形状，其它 6 个形状都计算这个分，求平均值。对于旋转对称的形状，比如 SZO 形，重复的形状不重复计分。


表示的是，在高度为 10（宽度为 10）的游戏池中，使用随机数种子 6，AI共放置了 30204 个方块，消除了 12073 行。要做多局统计的话，可以自己实现一个 python 脚本，多进程调用这个命令。

## 训练方法及计算量

本AI的参数，使用 CE 优化(Cross Entropy Method)，使用 20 samples，每个 sample 运行 100 局，每局跑高度13跑完整一局。迭代 100 轮，总计算量上限为：

```text
20 * 200000 * 100 * 100 = 40,000,000,000 行
```

上式里面的 200000 是平均消行数。在我的机器上(MacBook M1)，每小时能跑约 3.6 亿行，所以单个核心花费时间约为 110 小时，约5天不到的计算量。

### 单局的评估方法

取最终消行数 lines 为单局的分数。

### 多局的评分

对多组单局的分数，排序，然后去掉最高和最低的 1/3，剩下的取平均分作为最终分数。
